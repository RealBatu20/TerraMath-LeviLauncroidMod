# TerraMath — Levi Launchroid (Bedrock-native) conversion

A native Minecraft **Bedrock** (Android / Levi Launchroid) reimplementation of
[**TerraMath**](https://github.com/addavriance/TerraMath) by *addavriance* — a
Java mod that generates custom terrain from a user-supplied **math formula**
(e.g. `sin(x) * cos(z) * 10`). This project ports TerraMath's formula engine and
terrain model to C++ and exposes it through an on-device ImGui menu, building as
`libterramath.so` (arm64-v8a).

> Converted under the project's compliance rules: this is legitimate custom
> world generation — no cheats, exploits, or unfair advantage. EULA-safe.

### What is fully implemented and verified here
- **Formula engine** — recursive-descent parser + AST interpreter replacing
  TerraMath's Janino runtime compilation. All 50+ functions, constants, the `^`
  power operator, postfix `!` factorial, and implicit `lhs = rhs` equation mode.
- **Noise** — seeded Perlin / Simplex / octaved / composite implementations.
- **Terrain model** — exact port of `TerraMathDensityFunction` math (density +
  surface-height paths, smoothing, density mode, noise overlay).
- **Config** — JSON load/save (port of `ModConfig`).
- **ImGui menu** — formula editor, parameter sliders, noise controls, live
  surface preview, Apply/Save/Reset; GLES3 render via an `eglSwapBuffers` hook
  and touch input via an `AInputQueue` hook.
- **Build pipeline** — CMake (host tests + Android), `build.sh`, GitHub Actions.

These compile clean under `-Wall -Wextra` and pass **47 host unit tests**
(`./build.sh tests`).

### The one part that needs *your* binary
Bedrock has no Java-style density-function router. Terrain is generated by
native C++ in `libminecraftpe.so`, and the function to hook differs per game
version/ABI. Its **byte signature is not hardcoded** (shipping a guessed offset
would be fabricated and crash-prone). Instead you supply it via config
(`"terrainSignature"`); until you do, the engine and menu run normally but
generated terrain is unchanged. See **[docs/SIGNATURE_ANALYSIS.md](docs/SIGNATURE_ANALYSIS.md)**
for the exact IDA workflow to obtain it.

## Build

### Host tests (no Android tooling)
```bash
./build.sh tests
```

### Android `.so`
```bash
ANDROID_NDK_HOME=/path/to/android-ndk-r26d \
PRELOADER_INCLUDE=/path/to/levi-launchroid-sdk/include \
PRELOADER_LIB=/path/to/libgloss.a \
./build.sh android
# -> build-android/libterramath.so
```
Full details in **[docs/BUILD.md](docs/BUILD.md)**.

## Install & use
1. Copy `libterramath.so` into your Levi Launchroid mods directory.
2. Launch Minecraft. A small **TerraMath** button appears (top-left); tap it.
3. Type a formula, adjust sliders, press **Apply** (affects newly generated
   chunks), then **Save**.
4. To make terrain actually change, set `terrainSignature` in
   `…/games/com.mojang/terramath/terramath.json` (see signature docs).

Example formulas:
| Formula | Effect |
|---|---|
| `sin(x) * cos(z) * 10` | rolling waves |
| `sin(x/3) * 10 + cos(z/3) * 10` | broad swells |
| `sqrt(x*x + z*z)` | conical bowl |
| `perlin(x, 0, z) * 20` | smooth hills |
| `y = sin(x) * sin(z) * 30` | 3D iso-surface (equation mode) |
